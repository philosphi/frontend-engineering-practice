Array.prototype.reduce follows the Higher-Order Array Traversal skeleton but introduces an accumulator pattern absent from map and filter. An isInitialValue flag (computed via initialValue !== undefined) controls two things: whether the accumulator initializes to the provided value or the first array item, and whether the loop starts at index 0 or 1. On each non-hole iteration the accumulator is updated by passing it into the callback alongside the current item, index, and array — the return value becomes the new accumulator. The key TypeScript challenge was casting the no-initial-value branch: this[0] is typed as T but must be V, requiring an explicit as V cast. A subtle trap: checking initialValue || firstItem treats 0 as falsy and incorrectly bypasses a provided initial value of zero — the fix is !== undefined. Watch the ordering: isInitialValue must be computed before the empty-array guard can use it. A second attempt would be faster knowing the isInitialValue flag pattern, the as V cast rationale, and the !== undefined check upfront.
