Array.prototype.filter follows the same Higher-Order Array Traversal skeleton as map — for loop, i in this sparse check, callback invocation via .call() — with two key structural differences: the result array initializes to [] rather than new Array(this.length), and items are conditionally pushed rather than assigned by index. The most useful refactor was storing const item = this[i]! before the callback so the non-null asserted value is read once and reused in both the callback invocation and the push. The main difficulty was implementing thisArg: finding a meaningful test case (a plain object with a method, requiring a regular function so this isn't lexically captured), typing this as a fake first parameter in the callback (this: typeof filter), and typing thisArg as unknown since its shape is unknowable to the polyfill. A second attempt would be faster knowing the this parameter annotation syntax and the regular-vs-arrow distinction upfront.
